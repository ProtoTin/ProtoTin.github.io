Projects

Title: Banking System

Motivation of the project

The motivation for creating this banking system is to simulate a real-world banking environment in a simplified manner. The system is designed to handle basic banking operations such as account creation, deposits, withdrawals, and transfers. It includes classes to represent key entities such as the bank, customers, and different types of accounts.

Functional requirements/what the project does

The `Bank` class manages customers and generates unique IDs for customers and accounts. The `BankCustomer` class represents a customer with multiple accounts and the ability to create checking and savings accounts. The `BankAccount` class, an abstract class implementing the `Account` interface, provides a blueprint for different types of accounts, including methods for depositing, withdrawing, and transferring funds.

Project Design

The banking system project, written in Java and built with Gradle, applies the four pillars of object-oriented programming. It uses abstraction in the `Account` interface, encapsulation in the `BankAccount` and `BankCustomer` classes, inheritance in the `CheckingAccount` and `SavingsAccount` classes, and polymorphism through the `Account` interface. The project is developed feature by feature, with each feature being reviewed and revised based on partner feedback. The complexity of the project lies in its various features, such as account creation, deposits, withdrawals, transfers, and customer and ID management. The project is not a simple task, nor a minor tweak of a previous project, but a comprehensive banking system that applies the concepts learned throughout the semester.

Implementation

I implemented the Bank class, BankCustomer class, Customer interface, and the BankSimulation class.
